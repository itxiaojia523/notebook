    # 快速入门

    1. 下载和安装

2. 配置用户信息：用户名和邮箱
git config —global user.name “itxiaojia”
git config —global user.email  “chenminxin51@gmail.com”

远程仓库克隆到本地
git clone 地址

3. 拉取最新代码
git pull

4. 如果没有是新建项目 
git init

5. 拉去完代码，切换分支
#查看所有分区 标注*表当前所在分区
git branch

#创建新分区
git branch 分支名

#切换到指定分区
git checkout 分支

#创建并切换 
git checkout -b 分支名

#敲完代码后
git add. 一次性追踪所有文件，将文件加入暂存区

git commit -m "comment"

6. 推送代码
git push

本地仓库回滚:前提是没有提交给远程仓库
	git log //查看提交过的版本号
	git reflog -soft 版本号
	如果选hard就回到4.0版本，5.0代码不在
	如果选soft也回到4.0版本，但是修改的代码还在，只是还未提交的状态

    # git操作

    工作区文件四种状态
	1. untracked 未追踪 #不被git管理 
	2. 未被git管理 1<—>2 git add 文件名或.
	3. 已被git管理 未修改 已修改 已暂存

已精简方式检查文件状态
git status -s

红色？？表未追踪
git add追踪后 变绿 已追踪并且暂存
git commit -m “备注” #提交到本地仓库

红色M 表已修改 但未暂存 需要再次add 和 commit
绿色M 表已暂存
绿A 表新添到暂存区文件

#从暂存区移除文件
git reset HEAD 文件名

##⚠️ 撤销对文件的修改，还原成git仓库中保存的版本
git checkout —文件名

全部commit后
git status确认 nothing to commit

##移除文件
	1. 只从git仓库移除，保留工作区文件
git rm —cached 文件名
	2. git仓库和工作区同时移除
git rm -f 文件名

    # 远程分支操作

    #第一次推送分支
git push -u 远程仓库别名 本地分支名：远程分支名
-u表 关联本地和远程分支，只在初次使用，之后直接git push
远程仓库名 默认origin
ex.
git pish -u origin payment:pay

#查看远程分支
git remote show 远程仓库名称（origin）

#跟踪分支：即下载远程分支保存本地和远程分支名称相同
git checkout 远程分支名

#下载分支 并重命名
git checkout -b 本地分支名 远程仓库名:远程分支名
git checkout -b payment origin:pay



#删除远程分支
git push 远程仓库名 —delete 远程分支名

    # SVN 和 Git

    传统的版本控制系统如：SVN是基于差异的版本控制，它们存储的是一组基本文件和每个文件随时间逐步积累的差异

version1	version2	version3	version4	version5
FileA->		A1->					A2
FileB->					B1->					B2

好处：节省磁盘
坏处：耗时，效率低
每次切换版本，都需要在基本文件FileA上应用每个差异，从而生成目标版本对应的文件

Git的记录快照
在原有文件版本的基础上，重新生成一份新的文件，类似于备份
如果文件没有修改，Git不再重新存储该文件，而是只保留一个指针，指向之前存储的文件

version1	version2	version3	version4	version5
FileA		A1			A1			A2			A2
FileB		B			B			B1			B2
FileC		C1			C2			C2			C3

缺点：占用磁盘较大
优点：版本切换速度快，因为每个版本都是完整的文件快照
