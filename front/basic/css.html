    # flex布局

    	容器：父元素
	项目：子元素
	
	为父元素（容器）设置flex后，子元素（项目）的float clear vertical-align都失效
容器常见属性
	display：flex //flexible box的缩写 任何容器都可以指定（不区分块级和行内）
	flex-direction:设置主轴方向 默认x轴为主轴，主轴方向→
			row(默认x，→) 
			//reverse控制主轴方向
			row-reverse 
			column（y，↓） column-reverse
			元素跟着主轴排列
	justify-content：设置主轴方向的富裕空间
			默认flex-start从头部开始,center,space-around,space-between(两端贴边)	
	flex-wrap：是否换行
		默认不换行，如果超出，里面内容缩小 实现响应式
	
	align-items：侧轴上子元素的排列方式（单行）
		默认stretch; flex-start;center！
		stretch会把盒子拉的跟父元素一样高（用得少）
	
	align-content：侧轴上子元素的排列方式（多行）
		flex-start,center,space-around,space-between
	flex-flow：direction wrap 语法糖复合direction和wrap
	flex-flow: row wrap;

子项常见属性
	flex 子项占的分数 flex1/(flex1+flex2...) （不要设置宽度	）
	align-self 控制子项在侧轴的排列方式 
		默认auto 值为父元素的align-items 如果没有父元素 默认stretch
	order 默认0 值随意，根据order的值升序排列 
	如果没有设置flex-basis 默认拿width 或 height
	
flex-shrink：收缩因子，默认1   注意单行才有收缩

flex c2

flex版本
	1. display: box
	2. 旧版flex
	3. flex

flex-direction:水平row/垂直column -reverse（反向）

    # 三种传统布局方式

    1.普通流（标准流）：上下排列
  标签按照默认方式排列 如：块级元素独占一行；行内元素按照顺序 从左到右 遇到父元素边缘自动换行
2.浮动：：水平排列（浮动常和标准流配合排列）
最主要特点
1.脱离标准流
2.不再保留原先的位置
任何元素都可以浮动，添加后类似行内块元素
如果行内元素浮动了，不需要转为块或行内块就可以直接给高度和宽度
浮动后，大小由内容决定

为了约束浮动元素位置
1.采用标准流的父元素排列上下位置
2.内部子元素浮动排列

3.定位 ：自由移动或固定
1.相对：主要作用当爹 子绝父相  不脱离文档流，还占坑
2.绝对：默认视窗大小的初始..              脱离文档流
3.固定：没啥用，一般有绝对模拟固定  与绝对的区别：随着视窗移动  
定位到版心右侧：position left:50% margin-left:版心一半px

定位的盒子才有 z-index：正负0，越大盒子越靠上
一个盒子分两层（背景和文本），float半个盒子 定位提高一个盒子

给行内元素设置高度：转换，浮动，定位
块级元素设置定位:宽高默认内容大小
脱离标准流：不会触发外边距合并
浮动的元素不会压住下面标准流的文字，压住的是文字的盒子（文字环绕的效果）
浮动产生的目的：文字环绕图片

display :用于一个元素如何显示 block（块级之外，还有显示的意思） inline none(隐藏，位置也没了)
visibility ：用于指定是否可见 visible hidden(继续占有原来位置)
overflow : visible hidden scroll没超出也会出滚动条 auto 在需要的时候出现滚动条

    # 清除浮动

    why：因为父盒子在很多情况下不方便给高度，子盒子浮动了又不占有位置，最后父盒子高度为0，塌陷，影响到之后的标准流盒子
clear 本质闭合浮动，将影响限制在父盒子范围内
方法：
1.额外标签法：隔墙法，在浮动元素末尾添加空块级标签，例如div style=clear:both div；缺点添加无意义标签，结构化差
2.父级添加overflow：hidden auto scroll  外边距合并和清楚浮动，缺点无法显示溢出部分

3.父级添加after伪类：额外标签法升级版 最优！！！！！！！！！！！！！！！！！！！！！！！！
.clearfix:after{
	content:""
	display:block/table table的话好像不用height0了
	height:0
	clear:both
	visibility:hidden
}
.clearfix{
	zoom:1
}

4.父级添加双伪元素:同3，但是前后门都关上 最优！！！！！！！！！！！！！！！！！！！！！！！！！！
.clearfix:before,.clearfix:after{
	content:""
	display：table
}
.clearfix:after{
	clear:both
}
.clearfix{
	zoom:1
}

transform 转换，即变形，可以实现放大缩小，旋转，位移
属性：
1.translate(x,y)  translateX()=translate(x,0) 运用：鼠标经过图片，向上移动-5px
重点：
1.最大优点，不会影响到其他元素的位置
2.百分比相对自身元素：用于水平垂直居中 translate(50%,50%)
3.对行内无效

2.rotate(Xdeg) 正顺时针，负逆时针

3.transform-origin:x y 设置中心点 默认50% 50%（自身）
4.transform: scale(x,y) // 1,1 表原先一倍大小（不变）
用scale的优势：不会影响其他盒子，而且可以设置缩放中心点 

2D转化综合写法：
		transform:translate() rotate() scale()
	注意：顺序影响效果；位移最前面

    # 文档流和盒子模型

    文档流 normal flow
	-网页是一个多层结构
	-用户只能看到最上层
	-最底层称为文档流，是网页的基础
	-元素默认在文档流中排列
	
visibility: hidden 隐藏但依然占位置
元素类型display：table  / none隐藏元素 不占位置
	在文档流
		-块元素
			独占一行，垂直排列
			默认宽度父元素全部宽度，默认高度被内容撑满
		-行内元素
			自身大小，从左到右水平排列
		-行内块元素：可设置宽高 又不会独占一行 
			-2个行内块元素会有缝隙，相当于有空格
			<a …display:inline-block>
			<a …display:inline-block>
	不在文档流

盒子模型
块元素：
	内容区(content): width height
	内边距(padding)
	边框(border):边框宽度，边框颜色，边框样式  1px solid red
	外边距(margin)
	
	一个元素在父元素内，水平布局必须满足以下等式
	margin-left + border-left + padding-left + width + pading-right + border-right +margin-right = 父元素宽度
		如果以上值没有auto，默认按margin-right来调整
	三个能设置auto ： width margin-left margin-right
	width 默认auto 才会显得是100%
	
已知width 水平居中 margin: 0 auto

垂直方向布局
	如果子元素超过父元素高度，默认溢出 overflow: hidden / scroll 出现两条滚动条 /auto根据需求生成滚动条

行内元素的盒子模型
	不支持width height,设置padding,border,margin不会影响页面布局

案例：拖动模拟框
	1.点击登录后，弹出登录框，同时显示大的灰色背景

	var login = document.querySelector('.login')

    var link = document.querySelector('#link')
    var closeBtn = document.querySelector('#closeBtn')
    link.addEventListener('click',function (){
      mask.style.display = 'block'
      login.style.display = 'block'
    })
    closeBtn.addEventListener('click',function (){
      mask.style.display = 'none'
      login.style.display = 'none'
    })
    //鼠标拖拽：按下，移动，弹起
    //盒子位置：鼠标坐标-鼠标盒子内坐标
    var title = document.querySelector('#title')
    title.addEventListener('mousedown',function (e){
      //获取鼠标在盒子内坐标
      var x = e.pageX - login.offsetLeft
      var y = e.pageY - login.offsetTop
      document.addEventListener('mousemove',move)
      function move(e){
        login.style.left = e.pageX - x + "px"
        login.style.top = e.pageY - y + "px"
      }
      //鼠标弹起，移除移动事件
      document.addEventListener('mouseup',function (){
        document.removeEventListener('mousemove',move)
      })
    })
