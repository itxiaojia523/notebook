    # 跨域

    ajax默认遵守同源策略，违背同源策略就是跨域

##什么是同源
	协议，域名，端口都相同
##同源策略
	简单点来说：浏览器不允许A网站的js与非同源的网站C之间进行资源交互
	ex. 
		○ 无法读取非同源网页的 cookie localstorage indexdDB
		○ 无法接触非同源网页的DOM
		○ 无法发送Ajax请求( 跨域请求可以正常发起，浏览器也能接收到响应的数据，但是页面无法获取到，被浏览器拦截了)
		

正向代理：代理客户端
隐藏真实的请求客户端，通过代理服务器发送请求

反向代理：代理服务器
隐藏真实的服务器，通过代理相应请求

JSONP：非官方的跨域解决方案
	借助页面的script标签进行跨呀
	页面的一些标签本身具有跨域特性script link
	只支持get请求
	
CORS：cross-origin resource sharing
仅在开发时时候，上线时关闭
图片上传跨域上线时也需要cors


react中处理跨域

后端可以解决跨域

前端解决跨域
跨域问题：同源策略，能发请求，但是响应被ajax引擎拦截了
代理也是在3000接口，但是因为代理没有ajax，所以没问题

本质是：host3000请求host3000代理；host3000代理向后端发请求，再转发给host5000
## 

一般是请求地址应该是5000的，但是配置了前端代理，所以用3000就可以
http://localhost:3000 可以略

第一种
①
所有发给3000的请求都转发给了5000，只能配置一个代理
package.json中追加
"proxy":"http://localhost:5000"

注意：写axios时还是用3000，在3000有的资源不会去5000，没有的才会用5000

第二种
npm install http-proxy-middleware 默认下载好了
在src下新建setupProxy.js

const { createProxyMiddleware } = require("http-proxy-middleware");
module.exports = function(app) {
  app.use(
    createProxyMiddleware ("/api1", { //①如果请求地址有api1 就转发请求
      target: "http://localhost:5000", //②转发给5000
      changeOrigin: true,//③控制服务器收到的请求头的Host的值
      pathRewrite:{'^/api1':''} //④把请求中的/api1删掉，换成''
        //http://localhost:3000/api1/students  
    })，
    createProxyMiddleware ("/api2", { //①如果请求地址有api1 就转发请求
      target: "http://localhost:6000", //②转发给5000
      changeOrigin: true,//③控制服务器收到的请求头的Host的值
      pathRewrite:{'^/api2':''} //④把请求中的/api1删掉，换成''
        //http://localhost:3000/api1/students  
    })
  );
};

使用时：
//http://localhost:3000/api1/students  
//http://localhost:3000/api2/cars

react绑定事件
采用类似原生的第三种
注意：
	onClick 驼峰写法
	去掉();否则渲染的时候会执行一次

return <h1 onClick={demo}>今天的天气很{isHot?"炎热":"凉爽"}</h1>

onXXX 注意大小写
1. React使用的是自定义（合成）事件，而不是原生DOM
2. React中的事件是通过事件委托方式处理的（委托给组件最外层元素）

    高阶函数：满足以下任一条件
①：若A接受的参数是一个函数，那么可以称作高阶函数
②：若A的函数的返回值仍然是一个函数，那么可以称作告诫函数
常见的有：Promise      setTimeout      arr.map()

函数的柯里化：通过函数继续返回函数的方式，实现多次接受参数，最后统一处理的编码方式
