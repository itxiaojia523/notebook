    React路由

    # React路由

    # SPA理解

    SPA：single page application
单页Web应用
1. 整个应用只有一个页面
2. 点击页面的链接，不会刷新页面，只会做局部更新
3. 数据通过ajax请求获取，并在前端异步展示

    # 路由的理解

    一个路由就是一个映射关系key-value
key作为路径，value可能是function或component

    # 前端路由和后端路由

    前端路由：
1. 浏览器路由 value是component 用于内容展示
2. 注册路由<Route path="/test" component={Test}>
3. 流程
点击test链接按钮，url变成 xxx/test, 监控到url的变化，展示Test组件

路由根据浏览器BOM的history

history是栈结构

replace直接把栈顶的替换了

let history = History.createBrowserHistory() //方法1：直接使用H5推出的history的api
let history = History.createHashHistory() //方法2 hash值 锚点

后端路由：
1. value是function，用来处理前端提交的请求
2. 注册路由 router.get(path,function(req,res))
3. 流程
当node接到一个请求时，根据url找到匹配的路由，调用回调

    # 最佳实践

    1. 单独管理路由router.tsx

import {lazy, Suspense} from 'react'  //懒加载
import App from "App";
import {BrowserRouter, Routes, Route} from 'react-router-dom'
import Loading from 'components/Loading';
// app 和 缓存ui 不需要懒加载
// lazy(callback) callback中使用return import('Login')

interface Iroute {
    path: string,
    component: React.FC,
    children?: Iroute[]
}
const routerArr: Iroute[] = [
    {path: '/', component: App, children:[
        // 子路由可以不加/
        {path: 'list', component: lazy(()=> import('pages/List'))},
        // 不加/*  edit/1 跳转不过来
        {path: 'edit', component: lazy(()=> import('pages/Edit'))},
        {path: 'edit/:id', component: lazy(()=> import('pages/Edit'))},
        {path: 'means', component: lazy(()=> import('pages/Means'))},
        {path: 'namelist', component: lazy(()=> import('pages/NameList'))},
    ]},
    {path: '/login', component: lazy(()=> import('Login'))},
    {path: '/register', component: lazy(()=> import('Register'))},
]

const MyRouter = ()=>(
    <BrowserRouter basename='/cms_manage/'>
    {/* 注意位置在Routes外，必须要fallback属性：当在渲染时，使用loading组件 */}
        <Suspense fallback={<Loading/>}>
            <Routes>
                {
                    routerArr.map((item,index)=>(
                        item.children ?
                        // 如果有子路由 一般就2级，最多3级 再套一层就是了
                        <Route key={index} path={item.path} element={<item.component/>}>
                            {
                                item.children.map((item,index)=>(
                                    <Route key={index} path={item.path} element={<item.component/>}></Route>
                                ))
                            }
                        </Route>
                        :
                        // 没有子路由
                        <Route key={index} path={item.path} element={<item.component/>}></Route>
                    ))
                }
           
            </Routes>     
        </Suspense>
    </BrowserRouter>
)
export default MyRouter;

    # React-Router v6 新特性

    https://juejin.cn/post/7033313711947251743

    React Router 有以下三个不同的包放在npm上

1. react-router: 路由的核心库，提供很多组件和狗子

2. react-router-dom:包括react-router所有，并添加一些专门用于DOM的组件，比如<BrowserRouter>

3.react-router-native 包括react-router所有内容，并添加一些专门为ReactNative的API 如<NativeRouter>

新增多个hook
useParams 返回当前参数，根据url读取参数
useNavigate 代替useHistory
useMatch 
useOutlet 返回根据路由生成的element
useLocation 返回location对象
useRoutes 同Routers组件
useSearchParams 用于匹配URL中?后面的搜索参数

{* path是相对的，可以省略/  同时/表当前路径 *}
https://juejin.cn/post/7025418839454122015#heading-6  重点

{* 默认exact *}

{*  v6 Route 的 children 是用于嵌套路由,
嵌套的 parent route 的 path 不用加*
非嵌套的，分散在子组件的话，要加/* *}

    function App() {

    <div className="App">

        {/* switch 单一匹配 被Routes代替  路由写法改变*/}

}
